Sarathi process

Two examples show how artifact status expands from intent into code and executable evidence.

Open live project status
Background = artifact type Spec Design Plan Code + tests
Level tag = work scope Product Feature Slice Status badge = workflow state
WORK-* = parent allocation WORK-EXAMPLE Parent plan → Child artifact chain

Only code-ready leaves invoke /code-create. Those leaves may implement test obligations owned by any ancestor.

1. PR-sized leaf

No decomposition is needed.

Integration is incremental: slices prove boundaries, features prove local composition, and explicit integration slices prove cross-feature behavior. Traceability maps ownership; execution results prove outcomes.
