Agent-Steered SDLC: One Tree

Example product: a Facebook-style social app. Follow the arrows: each edge names the command that turns one artifact into the next.

Prompt: "Build a social app where users can create profiles, post updates, react, comment, and see a personalized feed."
Blue = Product/System Amber = Feature/Component Green = Slice/Change Purple = PR + code + tests

Example Artifact Tree

How to read the tree

Three levels are visible.

Product/system artifacts decompose into feature/component artifacts, which decompose into slice/change artifacts.

Readiness changes by level.

Product and feature artifacts can be valid while decomposable. Slice artifacts should become code-ready.

Tests land at the right place.

AT- criteria start in specs. Design names executable TEST- obligations and oracles. Plans assign both to PRs; code writes them and may add scoped inner tests discovered during TDD.

Spikes stay lightweight.

Exploratory prototypes use a timeboxed note with evidence and disposal criteria, not production-ready artifacts.

What each level carries

Scope Spec Design Plan
Product/System Mission, stakeholders, boundary, needs, non-goals, capabilities, docs/build/deploy expectations, major NFRs, broad ATs. HLD: context, major services/modules, drivers, boundaries, data ownership, docs/build/release/deploy strategy, ADRs, risks. Breakdown: milestones, feature/component work, dependencies, child artifacts, docs/build/deploy tracks, parallel tracks.
Feature/Component Parent refs, local behavior, FR/NFR/ATs, edge cases, rules, docs/build/deploy constraints, dependencies, non-goals. Responsibilities, contracts, local state/flows, core/shell split, docs/build/deploy impacts, decisions, TEST- obligations. Child slices or PRs, integration order, AT-/TEST- allocation, docs/build/deploy allocation, touch-scope risks.
Slice/Change Exact requirement delta, refined parent IDs, changed/unchanged behavior, docs/build/deploy delta, code-ready ATs. LLD: local changes, API/schema/data deltas, failure paths, docs/build/deploy scripts or artifacts, migration/rollback, TEST- obligations. Implementation PRs, Planned Touch Sets, Red/Green steps, AT-/TEST- allocation, docs/build/deploy verification, quality gates, worktrees.
